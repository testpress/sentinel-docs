# Backend Integration

Sentinel is designed to be started from your backend. Your server owns the Sentinel
API token, creates sessions, generates session tokens, and returns only the
frontend-safe values needed by the browser.

The browser should never create Sentinel resources directly. It should receive a
`sessionId`, `token`, and `orgCode` from your backend, then pass them to the SDK.
